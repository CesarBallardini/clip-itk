$FUNCNAME$
	COM_DTR()
$CATEGORY$
  Serial I/O
$SUMMARY$
	Queries/sets the Data Terminal Ready (DTR) status.
$LANG_RU$
	Запрашивает/устанавливает сигнал Data Terminal Ready (DTR).
$SYNTAX$
	COM_DTR(<nHandle>,[<lNewDTRStatus>]) --> lOldDTRStatus
$ARGUMENTS$
	<nHandle> is the file handle obtained from COM_OPEN().
	<lNewDTRStatus> - Designates the status of DTR. .T. designates
	that the DTR signal is active, and .F. designates that the signal is
	inactive. If the parameter is not specified, does not affect the DTR
	exit signal.
$LANG_RU$
	<номер обраб. файла> - это номер обрабатываемого файла, полученного с
	помощью функций COM_OPEN().
	<lNewDTRStatus> - Определяет статус DTR: .T. для активации
	сигнала, .F. для сброса сигнала. Если параметр не задан, статус DTR не
	изменяется.
$RETURNS$
	The returned value corresponds to the DTR signal status prior to
	the new setting, or in the absence of <lNewDTRStatus>, the returned
	value corresponds to the current DTR. A return of .T. indicates that the
	DTR is active (MCR bit 1 = 1). A return of .F. indicates that the DTR is
	inactive (MCR bit 1 = 0).
$LANG_RU$
	Возвращается .T. если DTR активен, .F. если нет. Если задан
	параметр <lNewDTRStatus>, возвращается значение сигнала перед вызовом
	функции.
$DESCRIPTION$
	COM_DTR() queries the DTR signal status.
	If the second parameter is not supplied, the signal status does not change.


	The COM_CLOSE() function resets DTR (to inactive). By contrast,
	COM_OPEN does not automatically make DTR active!

$LANG_RU$
	COM_DTR() запрашивает или устанавливает статус сигнала DTR.
	Если второй параметр не указан, статус сигнала не изменяется.

	Функция COM_CLOSE() сбрасывает сигнал DTR. Однако, COM_OPEN не
	устанавливает DTR автоматически.
$EXAMPLES$
	nHandle  :=  COM_OPEN("/dev/ttyS0")	       // Number of handle port
	IF nHandle > 0
		COM_DTR(nHandle, .T.)       // Activate DTR Port 1
		* ...
	ENDIF
